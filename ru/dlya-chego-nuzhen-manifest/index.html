Для чего нужен Манифест

~4 мин.

Файл Manifest

Файл Манифеста является частью технологии Web App Manifest - одним из ключевых атрибутов прогрессивного веб приложения. Наличие данного файла указывает браузеру на возможность установки веб-сайта как приложения с возможностью добавлять на главный экран и запускать без доступа к Интернету.

Файл манифеста включает в собя настройки представленные в формате JSON. Сам файл подключается на загруженной HTML странице через тег link. 

<link rel="manifest" href="/manifest.json">

Для каждой странице сайта можно подключить уникальный файл манифеста, указывающий на конкретную страницу, которая будет установлена как PWA.

Свойства Манифеста

В Манифесте можно настроить следующие свойства:

background_color

Определяет цвет фона, который браузеры могут использовать для стилизации за пределами окна страницы. Данный цвет может использоватся во время загрузки приложения, добавленного на главный экран.

Пример значенияОписание
#000Установит цвет фона как чёрный

categories

Массив ключевых слов, отражающих тематику веб-приложения. Браузеры могут использовать предоставленную информацию по своему усмотрению.

Пример значенияОписание
["игры", "музыка"]Предоставит браузеру информацию по тематике веб-приложения

description

Поле, описывающее предназначение веб-приложения.

dir

Определяет направление, по которому отображаются элементы манифеста, такие как name, description и другие.

Пример значенияОписание
autoНаправление будет зависеть от браузера
ltrСлева направо
rtlСправа налево

display

Определяет режим отображения для пользователя. Позволяет скрывать элементы браузера такие как поисковая строка, различные меню, закладки и так далее.

Пример значенияОписание
fullscreenСкрывает все элементы пользовательского агента
standaloneНекоторые элементы пользовательского агента будут скрыты
minimal-uiИсключается большинство элементов пользовательского агента
browserПриложение будет открыто в новой вкладке браузера

icons

Содержит массив объектов, определяющих параметры подключаемых изображений. Данные изображения отображаются во время загрузки приложения.

Каждый объект имеет следующие параметры:

ПараметрПример значенияОписание
srcicon.jpegПуть, по которому можно получить изображение
sizes96x96Разрешение изображения
typeimage/jpegMIME-тип изображения
purposebadge maskable anyОпределяет для каких целей используется изображение. Может быть использовано для нестандартных пользовательских агентов. По умолчанию стоит значение any - контекст изображения может быть любым

lang

Указывает на язык, используемый при составлении манифеста.

name

Определяет полное название веб-приложения.

short_name

Определяет краткое (сокращённое) название веб-приложения.

orientation

Определяет ориентацию, в которой приложение будет отображаться.

prefer_related_applications

Логическое значение указывающее браузеру на желательную установку приложения из параметра prefer_related Если указано значение true - браузер предложит пользователю установить нативное приложение из магазина приложений.

prefer_related

Представляет массив объектов, указывающих на магазины приложений, в которых располагается нативная версия данного веб-приложения.

Содержит следующие параметры:

СвойствоОписание
platformУказывает на магазин приложений, в котором располагается нативное приложение
urlАдрес, по котоорому можно найти приложение
idИдентификатор приложения в данном магазине

scope

Представляет собой область, в которой осуществляется навигация, пока манифест не будет загружен.

screenshots

Определяет массив объектов, ссылающихся на снимки, демонстрирующие работу веб-приложения.

Объекты в массиве аналогичны объектам из свойства icons

theme_color

Содержит цветовое значение, которое используется для стилизации темы окружения.

start_url

Свойство указывает на Url адрес, который будет использоваться при запуске приложения. Благодаря этому свойству можно на одном домене держать несколько веб-приложений, разделённых по url и по разным манифестам.

Поддержка браузерами

Некоторые из приведённых свойств могут не поддерживаться какими-либо браузерами, однако такое следует ожидать от новых технологий. За технологией Web App Manifest стоят широкие перспективы и возможности.
